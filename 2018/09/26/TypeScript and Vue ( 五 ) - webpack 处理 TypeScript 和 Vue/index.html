七，使用 webpack 构建 TypeScript and Vue 应用

  • 为什么开始使用 TypeScript

    1. TypeScript 的静态检查可以起到一定避免出低级bug的问题，更好的集中在编码的逻辑上
    2. 接口规范（后续更新）

  • add package.json scripts

  • add webpack.ts.vue.conf.js

  • add src/app-ts-vue/**

    • add src/app-ts-vue/index.ts

      • add vue-shims.d.ts 文件，和入口文件在同一层或者在子层下,如：./src/app-ts-vue/typings/vue-shims.d.ts

      作用：通过 import xx.vue 文件的时候不会有错误提示

      • ./src/app-ts-vue/app.vue

    • modified tsconfig

      • node 的作用：解析策略试图在运行时模仿Node.js模块解析机制

  • 注意：

    1. *.d.ts 文件中的字符必须使用 “” 否则，在项目文件中 我们指定的文件后缀的文件会提示：Can’t find this module
    2. 在 Vue 项目使用了 TypeScript 之后 babel 不能使用，会出现 errror（我在测试的时候情况是这样-@.@-）
    3. 在搭建工程时如果使用 html-webpack-plugin 这个插件记得在启动 dev 或 build 脚本中检查一下配置
