事务

  • 事物就是：需要在同一个处理单元中执行的一系列更新处理的集合

    同一处理单元：同一个数据库，同一个表中

  • 创建事务(MySQL)

    SQL 标准中是没有规定开始指令，根据 DBMS 的不同而不同，实际上几乎所有的数据库产品无需开始指令
    上面的是 MySQL 事务的操作

  • 事物的区分，有两种情况

    1. 每一条 SQL 语句就是一个事务（自动提交模式）
    2. 知道用户执行 COMMIT 或者 ROLLBACK 为止算作一个事物

  • 例子：

    上面的例子发生的事情：

    1. 创建了 ProductTest TABLE ,并设置 列的配置
    2. 使用 START TRANSACTION 语句告诉 MySQL 数据库我们开始一个事务
    3. 插入一条数据
    4. 做一次 ProductTest TABLE 的查询
    5. 这里示范 ROLLBACK ,取消事务的操作
    6. 最后一句查询 ProductTest TABLE 是验证 ROLLBACK 是否生效

    如果 不使用 ROLLBACK ，而是 COMMIT 操作就不一样了，插入的数据就会更新到 ProductTest TABLE 中

事务的特性（ACID）

  • 原子性(ATOMICITY)：每一条 SQL 语句都会按顺序执行
  • 一致性(Consistency)：出错的语句不会被执行，但是不会影响后面的语句（除非上一句 SQL 语句是下一句 SQL 语句的铺垫）
  • 隔离性(Lsolation)：事务之间是不可见的
  • 持久性(Durability)：在事务结束后改时间点的数据状态会被保存的特性

    保证持久性的方法：常见的就是将事务的执行记录保存到硬盘等介质中（该执行记录称为 ”日志“）

聚合

  • 聚合：就是将多行汇总成一行
  • 聚合的方式：
    • 使用聚合函数（COUNT, SUM, AVG …）
    • 使用分组子句（GROUP BY）
    • 使用子句(HAVING)

聚合函数

  • 聚合函数（聚集函数）：用于汇总的函数

    COUNT: 计算表中的记录条数（行数）

GROUP BY 子句

  • 在 GUROUP BY 子句中指定的列称为聚合键或者分组列

  • 和 SELECT 结合使用，子句的顺序（不能改变也不能替换）：

      1. SELECT – FROM – WHRER – GROUP BY

    • 分组会也会把 NULL 挑选出来为一新列

####

  • SELECT – FROM – GROUP BY 例子：

    将 purchase_price 作为 聚合键（分组列），展示出它们的总数

  • SELECT – FROM – WHRER – GROUP BY 例子：

    有 WHERE 子句先过滤掉对应的数据
    在将 purchase_price 作为 聚合键（分组列），展示出它们的 product_type 和 COUNT 的结果

使用 GROUP BY 时注意的地方

  • 使用 GROUP BY 子句时，SELECT 子句中不能出现聚合键（分组列）之外的列名
  • 不能使用 alias（别名）
  • 在 SELECT 子句中没有指明排序的方式，数据的展示时无序的
  • 在 WHERE 子句中使用 聚合函数

数据类型 (Data Type)
  • INTEGER：存储整数的数据类型（无法存储小数）
  • CHAR：（定长字符串）
    1. 在括号中指定该列可以存储的字符串的长度（最大长度），超过的部分无法输入到该列中
    2. 若存储的长度没有到达指定的长度，剩余的长度使用 半角空格保存
    3. 例子，Data Type: CHAR(6)，若数据写入只有 ‘james’，实际在数据库中的数据：’james ‘

  • VARCHAR：（可变字符串）

    1. 在括号中指定该列可以存储的字符串的长度（最大长度, 超过的部分无法输入到该列中??）
    2. 输入写入的长度就是实际存储的长度
    3. 例子，Data Type: CHAR(6)，若数据写入只有 ‘james’，实际在数据库中的数据：’james’

  • DATE：用来指定存储日期（年月日）的列的数据类型（日期型）

列的约束
  • 目前所知道的约束：
    1. NOT NULL: 代表不能输入空白（必填）
    2. NULL: 代表空白，可以不输入（非必填）
