诺曼门是啥

好的设计具备的两个重要的特征

  1. 可视性（discoverability）

    所设计的产品能不能让用户明白怎么操作是合理的，在什么位置及如何操作

  2. 易通性（understanding）

    所有设计的意图是什么，产品的预设用户是什么，所有不同的控制和装置起到什么作用

以人为本的设计与专业设计的角色区分

  1. 体验设计（注重情感在设计上的影响）

    设计产品，流程，服务，以及事件和环境的实践，重点关注整体体验的质量和愉悦感

  2. 工业设计（注重外形和材料）

    是一种专业的服务，为使用者和生产者双方的利益而创造和开发产品与系统的概念和规范，旨在优化功能，价值和外观

  3. 交互设计（注重易懂性和易用性）

    重点关注人与技术的互动，目标是增强人们理解可以做什么，正在发生什么，以及已经发生什么，交互设计借鉴了心理学，设计，文艺和情感等基本原则来保证用户得到积极的，愉悦的体验

  4. 以人为本的设计

    为了确保设计符合潜在用户需求和能力的一种设计流程

工程师的思维和设计师的思维的区别

  1. 工程师的思维侧重于产品的实现功能细节的逻辑，擅长使用技术去实现特定的功能（从机器的角度去考虑，产品的功能）

  2. 设计师的思维侧重于需要在技术和心里学中把握平衡

以人为本的设计

  1. 目的：提高产品的可视性和易通性

  2. 如何实现这种设计

交互设计的原则

认知和情感，会形成体验，良好的体验是如何形成的👇

交互设计主要是提高产品的可视性，原则（前5个是基本心理学概念）如下

  1. 示能(反示能)

    示能是一种关系，是一种能够展现产品属性/能力和使用主体的能力之间形成的关系（决定可能进行哪些操作）

    举个例子，比如一台笔记本电脑，功能很强大，可以做表格/剪视(音)频/做设计/玩游戏/用于开发，他的作用的分类也是区别于使用的人群，设计师可以使用制定的软件做图标/动效，工程师可以使用编辑器进行软件开发，小孩可以用于看电影/玩游戏等等

  2. 意符

    示能的符号提示功能叫做意符（点明操作的位置）

  3. 约束

    反示能

  4. 映射（Mapping）(数学概念)

    表示两个事物之间的关系

  5. 反馈（Feedback）（控制论和信息论的概念）

    沟通行动的结果

好的反馈是及时的，理解起来是简单的

  1. 系统的概念模型（Conceptual Models）

    概念模型通常是高度简化的说明，告诉你事物是如何工作（不必准确和完整）

系统映像

人们会对自己，他人，周围环境以及与他们互动的物品形成心理模式，这些心理模式是从经验，培训和指导中形成的概念模型，这些模型可以帮助我们实现目标和理解世界的指南

  1. 设计者概念模型 -> 系统映像（使用守则） -> 使用者概念模型（使用物品，可能在使用前阅读使用守则）
  2. 良好沟通是建立良好的概念模型的关键

存储过程

  1. 概念：

    • 可以将一些固定的操作集合起来，由数据库服务器来完成，实现某个特定任务
    • 存储过程是一个可编程的函数，同时可看作是在数据库编程中面向对象方法的模拟，它允许控制数据的访问方式

  2. 存储过程的优点

    • 可增强 SQL 语言的功能和灵活性
    • 良好的封装性
    • 高性能
    • 可减少网络流量
    • 存储过程可作为一种安全机制来确保数据库的安全性和数据的完整性

存储函数和存储过程的相同点/不相同点

  • 相同：

    1. 存储函数和存储过程，都是由 SQL 语句和过程式语句所组成的代码片段
    2. 并且可以被应用程序和其他 SQL 语句调用

  • 不相同：

    1. 存储函数不能有输出参数（因为存储函数自身就是输出参数），而存储过程可以拥有输出参数
    2. 可以直接对存储函数进行调用，且不需要使用 CALL 语句，而对存储过程的调用，需要使用 CALL
    3. 存储函数中必须包含一条 RETURN 语句，而这条特殊的 SQL 语句不允许包含于存储过程中

复习：模式和外模式

  • 在数据库的三级模式中，模式是数据库中全体数据的逻辑结构（对应关系数据库中的基本表）
  • 外模式：当不同的用户需要基本表中不同的数据时，可以为这些不同的用户建立不同的外模式（对应关系数据中的视图）
  • 模式和外模式之间关系之一：
    • 外模式的内容来源于模式，它是模式的部分数据或重构数据

视图

  • 概念：

    • 视图是数据中的一个对象，它是数据库管理系统提供给用户的以多种角度观察数据库中数据的一种重要机制
    • 视图是一个或多个表或者其他视图中通过查询语句导出的表
    • 视图和表十分相似，他包括若干数据列和数据行，并有自己的视图名，可通过 SQL 语句对视图进行创建，查询，更改，删除操作

  • 视图和表之间的区别

    • 视图不是数据库中真实的表，而是一张虚拟的表，其结构和数据是建立在对数据库中真实表的查询基础上
    • 视图的内容：是由存储在数据库中进行查询操作的 SQL 语句来定义
    • 视图本身不存储数据，它所对应的数据实际是储存在视图所引用的真实表（基础表）中
    • 作用：
      • 集中分散数据，可以在一个或多个表的基础上建立视图
      • 简化查询语言
      • 重用 SQL 语句
      • 保护数据安全
      • 共享所需数据
      • 更改数据格式

GROUP BY 子句

  • 作用： 将结果集中的数据行根据列的值进行逻辑分组

  • 语法：

    1. col_name 指定用于分组的选择列，可以指定多列，使用逗号分隔开，GROUP BY 子句中的各选择列必须也是 SELECT 语句的选择清单中的一项
    2. expr 指定用于分组的表达式
    3. position 指定用于分组的选择列在 SELECT 语句结果中的位置
    4. ASC | DESC：ASC 按升序分组，DESC（default） 按降序分组

  • 实践：

  • 注意

    • 使用 GROUP BY 子句的时候，列出的每一列必须是检索列或有效的表达式，但不能是聚合函数。如果在 SELECT 语句中使用列表达式，则必须在 GROUP BY 子句中指定相同的表达式

    • 除聚合函数之外，SELECT 语句中的每一个列都必须在 GROUP BY 子句中给出

    • 分组的列中包含 NULL 值，则 NULL 将作为一个单独的分组返回

HAVING 子句

  • 作用：用于过滤分组数据

  • 语法：

  • HVAING 和 WHERE 的区别

    • WHERE 子句主要用于过滤数据行，而 HAVING 子句主要用于过滤数据
    • HAVING 子句中的条件可以包含聚合函数，而 WHERE 子句中就不可以
    • WHERE 子句会在数据分组前进行过滤，HAVING 子句则会在数据分组后进行过滤
