复习：模式和外模式

  • 在数据库的三级模式中，模式是数据库中全体数据的逻辑结构（对应关系数据库中的基本表）
  • 外模式：当不同的用户需要基本表中不同的数据时，可以为这些不同的用户建立不同的外模式（对应关系数据中的视图）

  • 模式和外模式之间关系之一：

    • 外模式的内容来源于模式，它是模式的部分数据或重构数据

      视图

  • 概念：

    • 视图是数据中的一个对象，它是数据库管理系统提供给用户的以多种角度观察数据库中数据的一种重要机制
    • 视图是一个或多个表或者其他视图中通过查询语句导出的表
    • 视图和表十分相似，他包括若干数据列和数据行，并有自己的视图名，可通过 SQL 语句对视图进行创建，查询，更改，删除操作

  • 视图和表之间的区别

    • 视图不是数据库中真实的表，而是一张虚拟的表，其结构和数据是建立在对数据库中真实表的查询基础上
    • 视图的内容：是由存储在数据库中进行查询操作的 SQL 语句来定义
    • 视图本身不存储数据，它所对应的数据实际是储存在视图所引用的真实表（基础表）中
    • 作用：
      • 集中分散数据，可以在一个或多个表的基础上建立视图
      • 简化查询语言
      • 重用 SQL 语句
      • 保护数据安全
      • 共享所需数据
      • 更改数据格式

数据库 - SQL 和关系型数据库的基本操作（八）数据操作（DML）- 数据查询：GROUP BY 子句/分组数据 和 HAVING 子句

GROUP BY 子句

  • 作用： 将结果集中的数据行根据列的值进行逻辑分组

  • 语法：

    1. col_name 指定用于分组的选择列，可以指定多列，使用逗号分隔开，GROUP BY 子句中的各选择列必须也是 SELECT 语句的选择清单中的一项
    2. expr 指定用于分组的表达式
    3. position 指定用于分组的选择列在 SELECT 语句结果中的位置
    4. ASC | DESC：ASC 按升序分组，DESC（default） 按降序分组

  • 实践：

  • 注意

    • 使用 GROUP BY 子句的时候，列出的每一列必须是检索列或有效的表达式，但不能是聚合函数。如果在 SELECT 语句中使用列表达式，则必须在 GROUP BY 子句中指定相同的表达式

    • 除聚合函数之外，SELECT 语句中的每一个列都必须在 GROUP BY 子句中给出

    • 分组的列中包含 NULL 值，则 NULL 将作为一个单独的分组返回

HAVING 子句

  • 作用：用于过滤分组数据

  • 语法：

  • HVAING 和 WHERE 的区别

    • WHERE 子句主要用于过滤数据行，而 HAVING 子句主要用于过滤数据
    • HAVING 子句中的条件可以包含聚合函数，而 WHERE 子句中就不可以
    • WHERE 子句会在数据分组前进行过滤，HAVING 子句则会在数据分组后进行过滤

-通过 draggable 定义可拖拽的目标

  • 可拖动目标的是事件: dragstart drag dragend
    • dragstart：拖拽元素的开始的时候触发（在拖拽元素的一个生命周期只会触发一次）
    • drag：拖拽元素在拖拽的时候会反复触发（在拖拽元素的一个生命周期会反复触发）
    • dragend：拖拽元素的被鼠标释放的时候（在拖拽元素的一个生命周期只会触发一次）

查询操作

地位：数据查询是 SQL 语言的核心功能，也是数据库中使用最多的操作
作用：是从数据库中的一张或多张表（或视图）中检索出满足条件的数据信息

通常，查询的操作是由 0 行（没有满足条件的数据）或多行记录组成的一个记录集合，并允许选择一个或多个字段作为输出字段

数据理论基础：是关系数据模型中对表对象的一组关系，即选择，投影和连接

SELECT 子句

  • SELECT 语法中子句的使用次序以及说明
    • SELECT 要返回的列或表达式（必须使用）
    • FROM 从中检索数据的表（仅在表选择数据时使用）
    • WHERE 行级过滤（不必须使用）
    • GROUP BY 分组说明（不必须使用）
    • HAVING 组级过滤（必须使用）
    • ORDER BY 输出排序顺序（不必须使用）
